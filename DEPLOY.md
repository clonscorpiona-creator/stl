# Деплой STL платформы на Render

## Инструкция по развёртыванию

### 1. Подготовка проекта

1. Убедитесь, что все файлы закоммичены в Git:
```bash
git init
git add .
git commit -m "Initial commit"
```

2. Создайте репозиторий на GitHub и запушьте код:
```bash
git remote add origin https://github.com/your-username/stl-platform.git
git push -u origin main
```

### 2. Настройка на Render

1. Зайдите на [render.com](https://render.com) и войдите через GitHub

2. Нажмите **New +** → **Blueprint**

3. Подключите репозиторий с вашим проектом

4. Render автоматически распознает `render.yaml` и предложит развернуть:
   - Веб-сервис `stl-platform`
   - База данных PostgreSQL `stl-db`

5. Нажмите **Apply** для начала развёртывания

### 3. Настройка после деплоя

1. После первого развёртывания создайте суперпользователя:
```bash
# В панели Render перейдите во вкладку Shell
python manage.py createsuperuser
```

2. Проверьте, что медиафайлы загружаются корректно

### 4. Переменные окружения (автоматически из render.yaml)

- `DATABASE_URL` - подключается автоматически к PostgreSQL
- `SECRET_KEY` - генерируется автоматически
- `DEBUG=False` - производственный режим
- `ALLOWED_HOSTS=*` - разрешает все хосты

### 5. Дополнительная настройка

#### Домен
1. В панели Render перейдите в настройки сервиса
2. Добавьте собственный домен в разделе **Custom Domains**

#### Бэкапы базы данных
- Render автоматически делает бэкапы для бесплатного тарифа
- Для ручного экспорта: `pg_dump $DATABASE_URL > backup.sql`

#### Масштабирование
При росте трафика обновите тариф в настройках сервиса:
- От $7/мес для веб-сервиса
- От $7/мес для базы данных

### 6. Решение проблем

#### Ошибки миграции
```bash
python manage.py migrate --run-syncdb
```

#### Проблемы со статикой
```bash
python manage.py collectstatic --noinput --clear
```

#### Логи
Смотрите логи в панели Render во вкладке **Logs**

### 7. Безопасность

После деплоя:
1. Смените пароль суперпользователя
2. Настройте `ALLOWED_HOSTS` на ваш домен
3. Включите HTTPS (автоматически на Render)
