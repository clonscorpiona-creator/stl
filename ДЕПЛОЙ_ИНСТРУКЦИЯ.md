# 🚀 ДЕПЛОЙ STL CHAT НА RENDER

## ✅ Проект готов к деплою!

Все изменения закоммичены и отправлены в GitHub.

---

## 📋 ШАГ 1: Откройте Render

Перейдите на https://dashboard.render.com и войдите через GitHub.

---

## 📋 ШАГ 2: Создайте Web Service

1. Нажмите **"New +"** → **"Web Service"**
2. Нажмите **"Connect a repository"**
3. Найдите и выберите репозиторий: **`clonscorpiona-creator/stl`**

---

## 📋 ШАГ 3: Настройте сервис

Заполните поля:

| Поле | Значение |
|------|----------|
| **Name** | `stl-chat` |
| **Region** | Frankfurt (Germany) |
| **Branch** | `master` |
| **Root Directory** | (оставьте пустым) |
| **Environment** | `Python` |
| **Build Command** | `pip install -r requirements.txt && python manage.py collectstatic --noinput --clear && python manage.py migrate --noinput` |
| **Start Command** | `daphne -b 0.0.0.0 -p $PORT config.asgi:application` |

---

## 📋 ШАГ 4: Добавьте переменные окружения

Нажмите **"Advanced"** → **"Add Environment Variable"** и добавьте:

```
DEBUG = false
ALLOWED_HOSTS = *
CSRF_TRUSTED_ORIGINS = https://*.onrender.com
SECRET_KEY = stl-secret-key-production-2026-change-me
DATABASE_URL = sqlite:///db.sqlite3
PYTHON_VERSION = 3.14.3
```

---

## 📋 ШАГ 5: Выберите тариф

- **Plan**: Free
- **Instance Type**: Standard

---

## 📋 ШАГ 6: Запустите деплой

Нажмите **"Create Web Service"**

**⏱️ Время деплоя:** 3-7 минут

---

## ✅ ПРОВЕРКА ПОСЛЕ ДЕПЛОЯ

1. Откройте ваш URL: `https://stl-chat.onrender.com`
2. Проверьте:
   - ✅ Главная страница загружается
   - ✅ Вход/регистрация работают
   - ✅ Чат открывается
   - ✅ WebSocket подключается (зеленый индикатор)
   - ✅ Загрузка файлов работает

---

## 🔧 СОЗДАНИЕ СУПЕРПОЛЬЗОВАТЕЛЯ

После деплоя:

```bash
python manage.py createsuperuser
```

## 🆘 ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

### Ошибка сборки
- Проверьте логи: Dashboard → Service → Logs
- Убедитесь, что `requirements.txt` содержит все зависимости

### Ошибка запуска
- Проверьте Start Command
- Убедитесь, что daphne установлен (`pip show daphne`)

### Страница не загружается
- Подождите 1-2 минуты после деплоя
- Проверьте ALLOWED_HOSTS в переменных окружения

### WebSocket не подключается
- Убедитесь, что Start Command использует daphne
- Проверьте CSRF_TRUSTED_ORIGINS
