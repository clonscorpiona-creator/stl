  🔧 ИСПРАВЛЕНИЕ ОШИБКИ P3005 - База данных не пустая
═══════════════════════════════════════════════════════════

ПРОБЛЕМА:
Ваша существующая PostgreSQL база содержит таблицы.
Prisma не может применить миграции к непустой базе.

───────────────────────────────────────────────────────────

РЕШЕНИЕ:

1. Откройте настройки сервиса stl-scorpion161

2. Найдите раздел "Settings" (в левом меню)

3. Найдите поле "Build Command"

4. ЗАМЕНИТЕ текущую команду:
   npm install && npx prisma generate && npx prisma migrate deploy && npm run build

   НА ЭТУ:
   npm install && npx prisma generate && npx prisma db push --accept-data-loss && npm run build

───────────────────────────────────────────────────────────

ЧТО ИЗМЕНИЛОСЬ:

БЫЛО:    npx prisma migrate deploy
СТАЛО:   npx prisma db push --accept-data-loss

prisma db push - синхронизирует схему без проверки истории миграций

───────────────────────────────────────────────────────────

ПОСЛЕ ИЗМЕНЕНИЯ:

1. Нажмите "Save Changes"
2. Нажмите "Manual Deploy" → "Clear build cache & deploy"
3. Покажите мне скриншот новых логов
