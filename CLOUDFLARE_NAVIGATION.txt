## Пошаговая инструкция:

1. Откройте: https://dash.cloudflare.com

2. В левом меню найдите раздел "Workers & Pages"
   (иконка с молнией ⚡)

3. Вы увидите список ваших проектов
   Найдите проект "stl-platform" (или как вы его назвали)

4. Кликните на название проекта "stl-platform"

5. Вы попадёте на страницу проекта
   Сверху будут вкладки:
   - Deployments ← СЮДА
   - Settings
   - Analytics
   - Functions
   - etc.

6. Кликните на вкладку "Deployments"

7. Вы увидите список всех деплоев:

   ┌─────────────────────────────────────────────┐
   │ Production                                  │
   ├─────────────────────────────────────────────┤
   │ ❌ Failed  |  2 minutes ago  |  master      │
   │ ❌ Failed  |  10 minutes ago |  master      │
   │ ❌ Failed  |  20 minutes ago |  master      │
   └─────────────────────────────────────────────┘

8. Кликните на самый верхний (последний) Failed deployment

9. Справа вверху будет кнопка "Retry deployment"
   (или три точки ⋮ → Retry deployment)

───────────────────────────────────────────────────────────

## Если НЕТ проекта stl-platform:

Значит проект ещё не создан. Нужно создать новый:

1. Workers & Pages → Create application

2. Pages → Connect to Git

3. Выберите репозиторий: clonscorpiona-creator/stl

4. Begin setup

5. Заполните настройки (см. CLOUDFLARE_QUICK_START.txt)

───────────────────────────────────────────────────────────

## Почему упоминается Vercel?

Cloudflare Pages использовал адаптер @cloudflare/next-on-pages,
который внутри вызывает Vercel build. Это вызывало ошибки.

✅ ИСПРАВЛЕНО: Удалён wrangler.toml (commit 8c4e2f3a)

Теперь Cloudflare будет использовать стандартный Next.js build
без адаптера Vercel.
